## 3. Best Candidate for Implementation: **Silent Payments (BIP-352)**

### Research Status
- **Pros**: High score (13/15). Solves a major privacy gap for Bitcoin. TS logic (services/silent-payments.ts) is already mature.
- **Cons**: Native scanning in Kotlin/Rust is compute-intensive for mobile.

### Implementation Path
1. Replace `SilentPaymentManager.kt` stub with a native scanning engine.
2. Integrate a Rust-based EC worker for optimized point additions.
3. Bridge native scanning results to the TS `WalletViewModel`.

## 4. Secondary Candidate: **Ark V-UTXO PRF Refinement**

### Research Status
- **Pros**: Critical for deterministic sovereign recovery of off-chain funds.
- **Cons**: Requires strict alignment with `arkworks-rs` Blake2s implementation.

### Implementation Path
1. Integrate a native Blake2s implementation in `ArkManager.kt`.
2. Update V-UTXO deriving logic to use the proper PRF evaluate function.
