# Silent-payments JNI wrapper

This crate is the thin Android JNI boundary for the platform-neutral
`conxius-silent-payments` core. It is a `cdylib` and also exposes an `rlib` for host tests.

## Responsibilities

- Parse and validate BIP39 mnemonic/passphrase bytes inside Rust.
- Derive canonical BIP-352 receiver keys in Rust:
  `m/352'/coin_type'/account'/1'/0` and `m/352'/coin_type'/account'/0'/0`.
- Decode a bounded `SPB1` public transaction batch.
- Invoke the core scanner and encode only `SPR1` public match metadata/metrics.
- Map all failures to stable non-secret codes and catch panics at the JNI export.

The wrapper never logs, returns, serializes, or includes mnemonic, passphrase, seed, xpriv, child
secret, or tweak bytes. Secret buffers use best-effort zeroization. The only FFI-facing unsafe
boundary is the generated JNI export, which is kept in `src/lib.rs` and contains no scan logic.

## Host tests

```bash
cargo test --manifest-path native/silent-payments-jni/Cargo.toml --all-targets
```

Tests cover canonical BIP32 derivation, versioned codec round trips, stable error mapping,
truncated/malformed data, bounds, and an official BIP-352 vector public batch/result round trip.
They do not require a JVM or Android NDK.

Android outputs are generated by `scripts/build-silent-payments-android.sh` through the explicit
Gradle `buildSilentPaymentsNative` task. Generated libraries are not checked in.
